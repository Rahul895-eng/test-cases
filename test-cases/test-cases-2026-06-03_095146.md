## Requirement: REQ01

### [Positive] REQ01-TC01 — Clone settings to other discovery devices successfully

- **Pre-conditions:** A source discovery device is configured with all settings and features listed in the cloning table. At least one target discovery device is available on the network and reachable. Enhanced cloning (Clone to self) mechanism is enabled per FS 22.060.
- **Test Steps:**
  1. Log in to the source discovery device with administrator credentials.
  2. Navigate to the cloning configuration screen.
  3. Select 'Clone to self' (enhanced cloning) as the cloning method.
  4. Select the target discovery device(s) as the destination for cloning.
  5. Initiate the cloning operation.
  6. Wait for the cloning operation to complete and observe the status message.
  7. Log in to each target discovery device and inspect each setting and feature listed in the cloning table.
- **Expected Result:** The cloning operation completes successfully with a success status. All settings and features listed in the table are present and identical on the target discovery device(s) as configured on the source device.

### [Positive] REQ01-TC02 — Cloned settings preserved across an alt boot

- **Pre-conditions:** A discovery device has received all settings and features listed in the cloning table via the enhanced cloning (Clone to self) mechanism. The device supports alternate boot.
- **Test Steps:**
  1. Verify and record the current values of all settings and features listed in the cloning table on the cloned discovery device.
  2. Trigger an alternate boot (alt boot) of the discovery device.
  3. Wait for the device to fully complete the alt boot and become operational.
  4. Log in to the device after the alt boot.
  5. Inspect each setting and feature listed in the cloning table and compare against the recorded values.
- **Expected Result:** After the alt boot, all settings and features listed in the cloning table are preserved and match the values recorded prior to the alt boot.

### [Negative] REQ01-TC03 — Cloning fails when target device is unreachable

- **Pre-conditions:** A source discovery device is configured with all settings and features listed in the cloning table. The intended target discovery device is powered off or disconnected from the network. Enhanced cloning is enabled.
- **Test Steps:**
  1. Log in to the source discovery device with administrator credentials.
  2. Navigate to the cloning configuration screen.
  3. Select 'Clone to self' (enhanced cloning) as the cloning method.
  4. Select the unreachable target discovery device as the destination.
  5. Initiate the cloning operation.
  6. Observe the system response and status message.
- **Expected Result:** The cloning operation fails gracefully and displays a clear error message indicating the target device is unreachable. No partial or corrupted settings are applied, and the source device settings remain unchanged.

### [Negative] REQ01-TC04 — Non-cloneable settings are not transferred to target device

- **Pre-conditions:** A source discovery device is configured with settings both inside and outside the cloning table. A target discovery device with distinct non-cloneable settings (e.g., unique device identity/network address) is available and reachable. Enhanced cloning is enabled.
- **Test Steps:**
  1. Record the non-cloneable settings (settings not listed in the cloning table) on the target discovery device.
  2. Log in to the source discovery device and initiate enhanced cloning (Clone to self) to the target device.
  3. Wait for the cloning operation to complete.
  4. Log in to the target discovery device.
  5. Inspect the settings that are NOT listed in the cloning table and compare against the values recorded before cloning.
- **Expected Result:** Only the settings and features listed in the cloning table are transferred. The non-cloneable settings on the target device remain unchanged and are not overwritten by the source device's values.

## Requirement: REQ04

### [Positive] REQ04-TC01 — Set valid tray priority for each available tray

- **Pre-conditions:** Device is powered on and idle. User is logged in with administrator privileges. Trays 1-6 are installed and detected. User is at LUI Home screen.
- **Test Steps:**
  1. Navigate to Tools > Device Settings > Paper Management > Tray Settings.
  2. Select Tray 1 and open the Tray Priority setting.
  3. Assign priority value '1' to Tray 1 and confirm.
  4. Repeat for Trays 2 through 6, assigning unique ascending priority values 2 through 6.
  5. Tap Save/OK to apply the settings.
  6. Re-enter Tray Settings to verify the saved priority values.
- **Expected Result:** Each tray (1-6) accepts and retains its assigned unique priority value. The saved priorities are displayed correctly per tray after re-entering the settings, with no errors.

### [Positive] REQ04-TC02 — Tray priority governs automatic paper source selection

- **Pre-conditions:** Device is idle with administrator logged in. Trays 1 and 2 both contain the same paper size/type. Tray 2 is configured with higher priority than Tray 1.
- **Test Steps:**
  1. Navigate to Tools > Device Settings > Paper Management > Tray Settings.
  2. Confirm Tray 2 has a higher priority than Tray 1 and exit settings.
  3. Submit a print/copy job using Auto Tray Select for the common paper size.
  4. Observe which tray feeds the paper for the job.
  5. Empty Tray 2 and submit the same job again.
  6. Observe which tray feeds the paper.
- **Expected Result:** The first job feeds from the higher-priority Tray 2. After Tray 2 is emptied, the job feeds from the next priority tray (Tray 1), confirming priority ordering is honored.

### [Negative] REQ04-TC03 — Reject duplicate priority value across trays

- **Pre-conditions:** Device is idle with administrator logged in. Trays 1-6 are installed. User is in Tray Settings with each tray currently having a unique priority.
- **Test Steps:**
  1. Navigate to Tools > Device Settings > Paper Management > Tray Settings.
  2. Open Tray Priority for Tray 3 and note its value.
  3. Open Tray Priority for Tray 4 and attempt to set it to the same value as Tray 3.
  4. Attempt to save the configuration.
- **Expected Result:** The system prevents saving a duplicate priority value, displaying a validation error message and prompting the user to choose a unique priority. The previous valid values remain unchanged.

### [Negative] REQ04-TC04 — Reject out-of-range or invalid priority input

- **Pre-conditions:** Device is idle with administrator logged in. Tray 1 is installed and selected in Tray Settings. Valid priority range is 1-6.
- **Test Steps:**
  1. Navigate to Tools > Device Settings > Paper Management > Tray Settings.
  2. Select Tray 1 and open the Tray Priority setting.
  3. Attempt to enter an invalid priority value such as '0', '9', or a non-numeric/blank value.
  4. Attempt to save the configuration.
- **Expected Result:** The system rejects the invalid input, displays an appropriate validation/error message, does not save the value, and retains the previous valid priority for Tray 1.
